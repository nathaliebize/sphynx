Sphynx technical details

Overview

Sphynx is a web application that illustrate how users interact with websites. The tool allows you to follow your users’ behavior as they browse and interact through your pages.

The front-end is written in plain HTML and CSS, using Bootstrap to create a modern and responsive design. The HTML is rendered through Thymeleaf templates. A Java Spring boot application captures the REST requests on the back-end side. It processes and persists the data into a Postgres database using Hibernate as its ORM.

WEB SERVER

The server is configured with an Nginx reverse-proxy and a Tomcat application server. A Secure Socket Layer (SSL) certificate allows the server to receive secured requests. Nginx redirects the HTTP and HTTPS requests, received on ports 80 and 443, to the Tomcat webcontainer port. Tomcat enables a servlet based application to run.

DIGITAL OCEAN - DROPLET

Sphynx application is hosted on a DigitalOcean droplet, a web server running on Ubuntu. The memory of 1GB is scaled for our traffic’s flow and database’s size. The droplet is comprised of 1 virtual CPU, 1 TB of transfer and 25 GB of SSD disk. The cost of service is $5/month.

Why build sphynx?

They say that you learn the best by being hands-on. I created Sphynx to further my knowledge in writing a multi-tiered projet from scratch. I wanted to build a web application complex enough to involve technical areas such as security management, data collection and storage. I kept the requirements simple enough so that it could be built with realistic ressources in term of time, space and financial investment. Now that it's completed, I feel that it was a great experience to prepare myself for a career as a software engineer.

What did I learn along the way?

I learned the importance of writing clean and readable code. As I often had to go back to read the code I had written in the past, it became clear that investing in proper naming, small testable methods, and packaged structure made a huge difference in my ability to maintain the project.
I started my project with lots of spaggethi code. Over time, I learned that having a clear separation of concerns amongst my classes and application layers made things a lot easier to organize. I have several layers who have very specific responsibilities.
I learned to love unit tests. They helped me catch regressions for free as I was writing code. I wish I would have started writting tests earlier, it's much harder to play catch-up after a lot of code has been written.
I loved gaining an understanding of how everything was tied together, from the requests flow to the server code, database storage and presentation layer. I loved setting up my own server on a DigitalOcean droplet, configuring certificates and proxy servers to make everything work as I wanted.
