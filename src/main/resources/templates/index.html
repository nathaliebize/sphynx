SYNOPSIS

Grasps your user's experience!

You invested time and money building a website, polishing its design to make it intuitive for your customers. Yet, you still wonder if it has the impact that you desired. Sphynx allows you to understand how each user who lands on your website navigate through its contents.

Standard analytic tools focus on statistics and locations while Sphynx focuses on collecting quality and detailed session data. The tool is simple and free to use, I hope that it helps you fine-tune your websites so that they resonate with your users.

MOTIVE

Why build sphynx?

I created Sphynx to further my knowledge in writing a multi-tiered projet from scratch. I wanted to build a web application complex enough to involve technical areas such as security management, data collection and storage. I kept the requirements simple enough so that it could be built with realistic ressources in term of time, space and financial investment. I felt that it would be a good experience to prepare myself for a career as software engineer.

This is an open source project. It is meant to be transparent and accessible to all. The front-end is written in plain HTML and CSS, using Bootstrap to create a modern and responsive design. The HTML is rendered through Thymeleaf templates. A Java Spring boot application captures the REST requests on the back-end side. It processes and persists the data into a Postgres database using Hibernate as its ORM. You can find more details in the information section and access the entire project in Github.

What did I learn along the way?

I learned the importance of writing clean and readable code. As I often had to go back to read the code I had written in the past, it became clear that investing in proper naming, small testable methods, and packaged structure made a huge difference in my ability to maintain the project.
I started my project with lots of spaggethi code. Over time, I learned that having a clear separation of concerns amongst my classes and application layers made things a lot easier to organize. I have several layers who have very specific responsibilities.
I learned to love unit tests. They helped me catch regressions for free as I was writing code. I wish I would have started writting tests earlier, it's much harder to play catch-up after a lot of code has been written.
I loved gaining an understanding of how everything was tied together, from the requests flow to the server code, database storage and presentation layer. I loved setting up my own server on a DigitalOcean droplet, configuring certificates and proxy servers to make everything work as I wanted.
